# 论文进度追踪器 — 方案文档

---

## 一、执行方案 v1.0

### 项目结构

```
~/Projects/tool-paper-tracker/
├── tracker.html       ← 主文件，双击或浏览器打开
├── README.md          ← 使用说明
```

### 技术约束

- 纯 HTML + CSS + Vanilla JS
- 零外部依赖，零 npm，零 CDN
- CSS 变量做主题色
- 适配桌面浏览器（不专门适配手机，但不阻止访问）

### 数据模型

三类数据，存在浏览器 localStorage。

#### papers

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识，自动生成 |
| title | string | 论文标题 |
| status | enum | 写作中 / 投稿中 / R&R / 已接收 / 已发表 |
| journal | string | 目标期刊（可选） |
| deadline | string | 截止日期，ISO 格式 |
| totalSeconds | number | 累计工作时间（秒） |
| isRunning | boolean | 是否正在计时 |
| sessionStart | number | 本次计时开始时间戳（null 表示未在计时） |
| createdAt | string | 论文添加时间 |

#### reviews

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| journal | string | 期刊名 |
| deadline | string | 截止日期，ISO 格式 |
| status | enum | 待审 / 审稿中 / 已提交 |
| note | string | 备注 |
| createdAt | string | 添加时间 |

#### sessions

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| type | enum | paper / review |
| targetId | string | 对应 paper.id 或 review.id |
| targetName | string | 论文名或期刊名（冗余，方便日志展示） |
| start | string | 开始时间 ISO |
| end | string | 结束时间 ISO（null 表示还在计时） |
| durationSeconds | number | 时长（停止时写入） |
| note | string | 备注，可事后编辑 |

### UI 布局

三区 + 顶部工具栏。

**顶部工具栏**：标题"论文进度追踪器"，右侧 📥 导入 JSON、📤 导出。

**区域一：我的论文**
- "+ 添加论文"按钮 → 展开表单
- 表格：论文名 | 状态标签（彩色圆点 + 文字）| 目标期刊 | 截止日 | 累计时间 | 操作
- 操作：▶ 开始 / ⏹ 停止 / ✏️ 编辑
- 截止日渲染：已过期 → 红色背景；7 天内 → 黄色背景；8-14 天 → 无特殊样式
- 计时互斥：点任一篇的开始，自动停止其他
- 实时跑表：HH:MM:SS，每秒刷新
- 累计时间：停止后显示 Xh Ym

**区域二：审稿任务**
- "+ 添加审稿"按钮 → 展开表单
- 表格：期刊 | 截止日 | 状态 | 备注 | 操作（✏️ 编辑 / 🗑 删除）
- 截止日飘红规则同论文
- 审稿没有计时功能

**区域三：工作日志**
- 时间倒序排列
- 每条：日期 | 类型标签（写作/审稿）| 时长 | 备注 | 编辑备注
- 自动生成：论文计时停止时自动创建
- 手动添加：未用计时器的工作

### 导出/导入

- 导出 CSV：三个 CSV 文件，浏览器下载
- 导入 JSON：合并到当前数据，不做去重，导入前弹窗确认
- 手动备份：导出完整 JSON，用户自行 git commit && push

### 边界情况

| 情况 | 处理 |
|------|------|
| 第一版不关闭弹窗提醒 | 页面内 toast 提示 |
| 浏览器崩溃/关闭，计时器开着 | 下次打开时补算已过时间 |
| 同时打开两个标签页 | 第二个检测到计时标记，提示用户 |
| 数据损坏 | 导入时校验，损坏数据跳过并提示 |
| 空列表 | 显示引导文案，引导按钮添加 |

## 二、设计方案

### 方向：Scholarly Workbench（学者工作台）

工具型页面，核心任务：一眼看清论文状态、一键开始计时、数据不丢。
设计味道像安静的书房，不像 SaaS 后台。

### 配色

| 角色 | 色值 | 用途 |
|------|------|------|
| 页面底色 | `#F5F4F0` | 冷调纸白 |
| 卡片白 | `#FFFFFF` | 三块内容区背景 |
| 主文字 | `#1E1E24` | 蓝调近黑 |
| 次文字 | `#6B6E7B` | 石板灰 |
| 强调色 | `#2F4985` | 深靛蓝 — 按钮、链接、计时器 |
| 强调悬浮 | `#1E3466` | 深一度靛蓝 |
| 分割线 | `#E4E3DD` | 微妙暖灰 |
| 过期红 | `#C0392B` | 暗红 |
| 过期红底 | `#FFEBEE` | 浅玫红底 |
| 临近黄 | `#B8860B` | 暗金 |
| 临近黄底 | `#FFF8E1` | 暖黄底 |

**五种论文状态色**：

| 状态 | 色值 | 含义 |
|------|------|------|
| 写作中 | `#B8860B` 暗金 | 正在进行，台灯暖光 |
| 投稿中 | `#4A7C96` 钢蓝 | 已寄出，在编辑部的信箱 |
| R&R | `#6B4C7A` 深紫 | 修改的混合信号 |
| 已接收 | `#3D7A5C` 森林绿 | 通过 |
| 已发表 | `#2E7D7B` 深青 | 尘埃落定，印成铅字 |

### 字体

零 CDN，macOS 优先系统字体：

| 角色 | 字体栈 |
|------|--------|
| 标题 | `Charter, Georgia, serif` |
| 正文/表格 | `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` |
| 计时器 | `"SF Mono", Menlo, Consolas, monospace` |

### 标志性元素：计时器行脉动光效

正在计时的论文行获得页面唯一的光效——微弱靛蓝色 box-shadow 脉动（3 秒周期）。
计时器数字用等宽字体 + `tabular-nums`，数字宽度固定不跳动。
这是整个页面唯一"动"的地方。

### 组件细节

- **状态标签**：彩色圆点 6px + 文字，小写，不加粗不加背景色块
- **截止日**：过期红底红字，7 天内黄底黄字，仅日期单元格变色
- **空状态**：引导文案（如"还没有论文，点击「+ 添加论文」开始追踪"），`#6B6E7B`，居中
- **按钮层级**：主要操作靛蓝实色，次要灰边白底，危险操作红字悬浮红底
- **表单**：点击按钮后在表格上方内联展开，`max-height` 过渡动画，非弹窗
